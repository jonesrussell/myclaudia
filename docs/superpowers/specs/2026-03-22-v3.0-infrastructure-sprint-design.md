## Design

### Shared Pattern

All new providers follow the established lazy-singleton pattern used by `CacheServiceProvider` and `TelescopeServiceProvider`:

```php
final class FooServiceProvider extends ServiceProvider
{
    private ?FooInterface $foo = null;

    public function register(): void
    {
        // Framework DI registration when resolver supports set()
    }

    public function getFoo(): FooInterface
    {
        if ($this->foo === null) {
            $this->foo = new ConcreteFoo(/* config */);
        }
        return $this->foo;
    }
}
```

Storage-backed providers use `storage/` directory with SQLite (WAL mode) and graceful fallback to in-memory on mkdir failure.

### #479 Cache: Wire EntityCacheInvalidator

**What:** Connect the existing `EntityCacheInvalidator` (already built in `CacheServiceProvider`) to the Symfony `EventDispatcher` so entity saves trigger tag-based cache invalidation.

**Event names:** `EntityEvents::POST_SAVE->value` (`waaseyaa.entity.post_save`) and `EntityEvents::POST_DELETE->value` (`waaseyaa.entity.post_delete`). The invalidator exposes `onPostSave(EntityEvent)` and `onPostDelete(EntityEvent)`.

**Where:** `ClaudrielServiceProvider` or a boot-phase hook where both `EventDispatcher` and `CacheServiceProvider` are available.

**Acceptance:** Entity save on any cached type (commitment, person, workspace) clears relevant cache bin entries.

### #478 Telescope: Wire Recorders

**What:** `TelescopeRequestMiddleware` already exists in `src/Middleware/`. The remaining work is wiring it into the kernel's request lifecycle via `ClaudrielServiceProvider::middleware()`. Also wire `CacheRecorder` to cache events if the cache package exposes event hooks.

**Blocked:** `QueryRecorder` and `EventRecorder` require framework hooks that don't exist yet in waaseyaa/database-legacy. Document this gap, don't work around it.

**Acceptance:** HTTP requests appear in telescope storage. Cache operations recorded when cache events are available.

### #480 State: SqlState Provider

**What:** Add `waaseyaa/state` to `composer.json` require (promote from transitive to direct dependency). Create `StateServiceProvider` with `SqlState`.

**Constructor:** `SqlState(DatabaseInterface $database)`. Requires `DatabaseInterface`, not raw PDO. The provider must obtain the existing `DatabaseInterface` from the service resolver (same instance used by entity storage).

**Interface:** `StateInterface` exposes: `get($key, $default)`, `getMultiple($keys)`, `set($key, $value)`, `setMultiple($values)`, `delete($key)`, `deleteMultiple($keys)`.

**Acceptance:** Provider instantiates without error. State round-trips (set/get, setMultiple/getMultiple, delete) in integration test.

### #481 Queue: SyncQueue Provider

**What:** Add `waaseyaa/queue` to `composer.json` require. Create `QueueServiceProvider` with `SyncQueue` (not `MessageBusQueue`, which requires Symfony Messenger). `SyncQueue` takes an optional `$handlers` array and `AttributeGuard`.

**Constructor:** `SyncQueue(array $handlers = [], ?AttributeGuard $guard = null)`

**Interface:** `QueueInterface::dispatch(object $message): void`

**Acceptance:** Provider instantiates. Dispatching a test message invokes the handler synchronously.

### #482 Mail: Mailer Provider

**Prerequisite:** `waaseyaa/mail` is not yet published as a split sub-package. Must run the monorepo split first (tag + "Split Monorepo" GitHub Action) before `composer require` will work. If not yet published, skip this issue and move it to the next sprint.

**What:** Add `waaseyaa/mail` to `composer.json` require. The package ships its own `Waaseyaa\Mail\MailServiceProvider` which registers `TransportInterface` and `MailerInterface` as singletons. Claudriel's provider wraps this (same pattern as `TelescopeServiceProvider` wrapping `WaaseyaaTelescopeServiceProvider`).

**API:** `MailerInterface::send(Envelope $envelope)` where `Envelope` is `Waaseyaa\Mail\Envelope` (not `Email`). Constructor: `Envelope(array $to, string $from, string $subject, string $textBody, string $htmlBody, array $headers)`.

**Transports:** `LocalTransport` (logs to file) and `ArrayTransport` (in-memory). No SMTP transport exists yet. Dev uses `LocalTransport` logging to `storage/mail.log`. Production SMTP is a future package addition.

**Acceptance:** Provider instantiates with `LocalTransport`. Sending a test envelope writes to log file.

## Testing Strategy

Each provider gets an integration test in `tests/Integration/Provider/`:

- `StateServiceProviderTest` - round-trip set/get
- `QueueServiceProviderTest` - synchronous dispatch
- `MailServiceProviderTest` - instantiation with null transport
- `CacheServiceProviderTest` - entity invalidation after save (extend existing test)
- `TelescopeServiceProviderTest` - request recording after middleware wiring
