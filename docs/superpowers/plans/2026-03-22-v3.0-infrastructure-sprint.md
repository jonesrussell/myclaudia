# v3.0 Waaseyaa Infrastructure Alignment — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Register five Waaseyaa infrastructure packages (state, queue, mail, cache, telescope) as first-class service providers in Claudriel.

**Architecture:** Each package gets a lazy-singleton provider following the established pattern (CacheServiceProvider, TelescopeServiceProvider). New packages are promoted to direct composer dependencies. Cache and telescope get their existing providers wired to the event system and request lifecycle.

**Tech Stack:** PHP 8.4, Waaseyaa framework packages, PHPUnit, Symfony EventDispatcher

**Spec:** `docs/superpowers/specs/2026-03-22-v3.0-infrastructure-sprint-design.md`

```php
<?php

declare(strict_types=1);

namespace Claudriel\Provider;

use Waaseyaa\Database\DatabaseInterface;
use Waaseyaa\Foundation\ServiceProvider\ServiceProvider;
use Waaseyaa\State\SqlState;
use Waaseyaa\State\StateInterface;

final class StateServiceProvider extends ServiceProvider
{
    private ?StateInterface $state = null;

    public function register(): void
    {
        // DI registration when resolver supports set()
    }

    public function getState(): StateInterface
    {
        if ($this->state === null) {
            $database = $this->resolveDatabase();
            $this->state = new SqlState($database);
        }

        return $this->state;
    }

    private function resolveDatabase(): DatabaseInterface
    {
        // Use in-memory SQLite for standalone instantiation (tests, CLI).
        // In the app kernel, the shared DatabaseInterface should be injected instead.
        return DBALDatabase::createSqlite(':memory:');
    }
}
```

Add the import: `use Waaseyaa\Database\DBALDatabase;`

Note: `resolveDatabase()` creates an in-memory SQLite for standalone use (matching the upstream `SqlStateTest` pattern). When registered in the app kernel, the provider should receive the shared `DatabaseInterface` instead.

<?php

declare(strict_types=1);

namespace Claudriel\Provider;

use Waaseyaa\Foundation\ServiceProvider\ServiceProvider;
use Waaseyaa\Mail\Mailer;
use Waaseyaa\Mail\MailerInterface;
use Waaseyaa\Mail\Transport\LocalTransport;
use Waaseyaa\Mail\Transport\TransportInterface;

final class MailServiceProvider extends ServiceProvider
{
    private ?MailerInterface $mailer = null;

    public function __construct(
        private ?TransportInterface $transport = null,
    ) {}

    public function register(): void
    {
        // DI registration when resolver supports set()
    }

    public function getMailer(): MailerInterface
    {
        if ($this->mailer === null) {
            $transport = $this->transport ?? $this->createDefaultTransport();
            $fromAddress = $_ENV['MAIL_FROM'] ?? getenv('MAIL_FROM') ?: 'noreply@claudriel.ai';

            $this->mailer = new Mailer($transport, $fromAddress);
        }

        return $this->mailer;
    }

    private function createDefaultTransport(): TransportInterface
    {
        $storageDir = dirname(__DIR__, 2).'/storage';
        if (! is_dir($storageDir) && ! mkdir($storageDir, 0o755, true)) {
            error_log('Mail: could not create storage/ directory');
        }

        return new LocalTransport($storageDir.'/mail.log');
    }
}
```

public function boot(): void
{
    // Wire EntityCacheInvalidator to entity lifecycle events.
    // Requires EventDispatcher to be available from the service resolver.
}

public function wireInvalidator(EventDispatcherInterface $dispatcher): void
{
    $invalidator = $this->getEntityCacheInvalidator();

    $dispatcher->addListener(
        EntityEvents::POST_SAVE->value,
        [$invalidator, 'onPostSave'],
    );

    $dispatcher->addListener(
        EntityEvents::POST_DELETE->value,
        [$invalidator, 'onPostDelete'],
    );
}
```
