# v1.2 Public Account Smoke Matrix

## Purpose

This matrix defines the minimum smoke surface for the public signup/account subsystem.

## Smoke Test Matrix

| Flow | Surface | Preconditions | Check | Expected result |
|---|---|---|---|---|
| Public signup | `GET/POST /signup` | App reachable | Submit name, email, and password | Pending account is created and verification delivery is queued |
| Email verification | `GET /verify-email/{token}` | Fresh signup exists | Open verification link | Account becomes active and onboarding redirect includes tenant/workspace context |
| Login | `GET/POST /login` | Verified account exists | Submit valid credentials | Session is created and redirect lands with tenant/workspace context |
| Password reset | `POST /forgot-password` then `POST /reset-password/{token}` | Verified account exists | Issue reset link and submit new password | Reset token is consumed once and credentials update |
| New-tenant dashboard/brief | `/` and `/stream/brief?transport=fallback` | Verified account has completed bootstrap | Load dashboard JSON fallback and brief fallback | Default workspace is visible and tenant-scoped first-run data renders |
| New-tenant chat | `GET /stream/chat/{messageId}` | Verified account and default workspace exist | Send a local workspace-creation instruction | Tenant-scoped chat local action succeeds without leaking out of tenant context |
| Workspace CRUD | `/api/workspaces` | Verified account and tenant context exist | Create, update, and delete a workspace | CRUD succeeds only inside the authenticated tenant scope |
