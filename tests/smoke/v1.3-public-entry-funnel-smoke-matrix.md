# v1.3 Public Entry Funnel Smoke Matrix

## Purpose

This matrix defines the minimum smoke surface for the marketing homepage and public-to-app journey.

## Smoke Test Matrix

| Flow | Surface | Preconditions | Check | Expected result |
|---|---|---|---|---|
| Marketing homepage | `GET /` | App reachable | Load the root page | Public homepage renders product framing plus signup and login CTAs |
| Anonymous app entry | `GET /app` | No authenticated session | Open app shell route directly | Request redirects to `/login` |
| Login to app | `GET/POST /login` | Verified account exists | Submit valid credentials | Session is created and redirect lands in `/app` with tenant/workspace context |
| Authenticated homepage re-entry | `GET /` | Authenticated session exists | Visit the public homepage while signed in | Request redirects to `/app` instead of rendering the anonymous funnel |
| Verification to app | `GET /verify-email/{token}` then `GET /onboarding/bootstrap` | Fresh signup exists | Complete verification and bootstrap | Ready accounts terminate in `/app` with verified tenant/workspace context |

## Staging Validation Shortcuts

- `curl -si https://claudriel.northcloud.one/` and verify homepage CTA markers
- `curl -si https://claudriel.northcloud.one/app` and verify `302` with `Location: /login`
- `curl -si https://claudriel.northcloud.one/login` and verify the public auth surface still renders
